# 项目说明文档

## 项目规划
本项项目是一个 AI Agent 聊天系统，包含前端 (webs) 和后端 (server)。后端基于 Express 和 LangChain 开发，提供对话接口。

## 实施方案
1.  **后端环境配置**：确保 `server/package.json` 正确配置 ES 模块支持。
2.  **依赖安装**：安装后端必要的依赖项。
3.  **服务器启动**：在 3000 端口启动后端服务。
4.  **前端连接**：确保前端能够正确访问后端接口。

## 数据库环境准备
1. **自动模式** (推荐)：
   - 确保本地 MySQL 已启动，且账号/密码为 `root` / `root`。
   - 在 `server` 目录下运行 `npm run db:setup`，系统将自动创建数据库。
2. **手动模式**：
   - 使用 Navicat 或 MySQL 命令行导入 [ai_agent_db.sql](file:///e:/wxwork/ai-agent-project/server/db/ai_agent_db.sql) 文件。

## 部署指南 (100% 免费方案)
作为简历作品集，推荐以下组合进行免费部署：

### 1. 后端部署 (Express Server)
*   **平台**：[Koyeb](https://www.koyeb.com/) 或 [Render](https://render.com/)
*   **步骤**：
    1.  将 `server` 文件夹推送到 GitHub。
    2.  在 Koyeb/Render 关联仓库，选择 `Node.js` 环境。
    3.  **环境变量**：设置 `PORT=3000` 以及 `OPENAI_API_KEY` 等。
    4.  **注意**：Render 免费版会有休眠，Koyeb 的 `Nano` 实例相对更稳定。

### 2. 前端部署 (Vue 3 Frontend)
*   **平台**：[Vercel](https://vercel.com/) (强烈推荐)
*   **步骤**：
    1.  将 `webs` 文件夹推送到 GitHub。
    2.  在 Vercel 导入项目，框架选择 `Vite`。
    3.  **环境变量**：添加 `VITE_API_URL`，值为你部署好的**后端地址**（例如 `https://your-api.koyeb.app`）。
    4.  Vercel 会自动完成构建并提供 HTTPS 域名。

### 3. 高级方案 (AI 作品集专用)
*   **平台**：[Hugging Face Spaces](https://huggingface.co/spaces)
*   **优势**：AI 圈的 GitHub，完全免费，非常适合展示 AI Agent 作品。可以使用 Docker 部署前后端。
